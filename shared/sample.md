### **슬라이드 2: AI 진화와 Companion Agent의 부상**

#### **🚀 AI 기술의 3단계 진화**

```
2023-2024: AI Assistant 시대
├─ 단순 작업 지원 (검색, 요약, 초안)
├─ 수동적 응답 (명령 대기)
└─ 컨텍스트 부족 (대화 독립)

         ⬇

2025: AI Agent 시대 (현재)
├─ 자율적 작업 수행 (멀티스텝)
├─ 목표 지향 행동 (Planning-Execution)
└─ 도구 사용 능력 (MCP Tools)

         ⬇

2025-2026: AI Companion 시대 ⭐
├─ 상시 동반자 (24/7 Knowledge Partner)
├─ 맥락 이해 (업무 히스토리 + 조직 지식)
├─ 능동적 지원 (Proactive Assistance)
└─ 지속적 학습 (Knowledge Update)
```

**Companion의 의미**
> "Com(함께) + Panion(빵을 나누다)"  
> 같은 경험을 공유하는 동료, 업무 전반에 걸쳐 함께하는 지식 파트너

### **슬라이드 3: 기업 내 지식 관리의 과제와 솔루션**

#### **💡 현실적 과제**

```
❌ 정보 과부하
   → 방대한 데이터 속에서 필요한 정보 찾기 어려움
   → 하루 평균 2.5시간을 정보 검색에 소비

❌ 지식 사일로
   → 부서별로 분산된 정보, 접근성 저하
   → 암묵지의 명시화 부재

❌ 저품질 챗봇 난립
   → "각 부서마다 Ollama + Naive RAG로 만든 
      저품질 챗봇들이 산재"
   → 높은 환각(Hallucination) 발생률
   → 통합 관리 불가능

❌ 지식 손실
   → 직원 이직 시 노하우 소실
   → 베스트 프랙티스 미축적
```

#### **✅ Companion Agent 솔루션**

```
✓ 조직 지식의 중앙화
  → 단일 플랫폼 통합 관리, 부서 사일로 제거

✓ 엔터프라이즈급 정확도
  → Hybrid RAG + CRAG 검증으로 환각 방지

✓ 플랫폼 네이티브 통합
  → LangGraph OAP 기반, MCP 프로토콜 연동

✓ 지속 가능한 지식 생태계
  → 실시간 업데이트, 세대 간 지식 전수 체계
```

### **슬라이드 5: 핵심 기능**

#### **1. 지능형 검색 (Hybrid RAG)**

```
🔍 3가지 검색 전략 조합
   • Semantic Search: 의미 기반 벡터 유사도
   • Keyword Search: 정확한 용어 매칭
   • Hybrid Search: 가중치 통합 + Re-ranking

📊 정확도 보장
   • CRAG 검증으로 환각 방지
   • 출처 기반 신뢰 가능한 답변
   • Multi-Query로 포괄적 정보 수집
```

#### **2. 문서 관리**

```
📄 Multi-Format 지원
   • PDF, DOCX, TXT, MD, HTML 자동 파싱
   • Semantic Chunking (512 tokens)
   • OpenAI Embeddings (1536 dim)

🗂️ Collection 기반 조직화
   • 부서/프로젝트별 지식 분류
   • 메타데이터 태깅 (작성자, 날짜, 카테고리)
   • 접근 권한 제어 (RBAC)
```

#### **3. 플랫폼 통합 (LangGraph OAP)**

```
🎨 No-Code Configuration
   • 웹 기반 비주얼 인터페이스
   • 실시간 에이전트 상태 모니터링

🤝 Multi-Agent Orchestration
   • Supervisor를 통한 작업 조율
   • 전문화된 에이전트 협업
```

### **슬라이드 6: Companion Agent 작동 프로세스 - 지식의 선순환**

#### **🔄 4단계 지식 생성 및 확장 사이클**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣ QUERY EXPANSION (쿼리 확장)
   
   사용자 질문
   └─→ multi_query tool
        └─→ 3-5개 서브 쿼리 자동 생성
             └─→ 다각도 정보 수집

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

2️⃣ HYBRID RETRIEVAL (하이브리드 검색)
   
   각 서브 쿼리별 검색
   ├─→ Semantic Search (의미 유사도)
   ├─→ Keyword Search (정확 매칭)
   └─→ Hybrid Fusion (통합 랭킹)
        └─→ CRAG 검증 (관련성 필터링)
             └─→ Top-K 문서 선정

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

3️⃣ GROUNDED GENERATION (근거 기반 생성)
   
   선정된 문서 기반
   └─→ LLM 답변 생성
        ├─→ 환각 방지 (Ground Truth)
        ├─→ 출처 명시 (인용)
        └─→ 피드백 수집
             └─→ 품질 평가 (👍/👎)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

4️⃣ KNOWLEDGE CREATION (지식 생성 및 저장) ⭐
   
   새로운 인사이트 발견
   └─→ create_collection tool
        └─→ 새로운 지식 Collection 생성
             ├─→ "트러블슈팅_사례집_2025"
             ├─→ "품질_개선_베스트프랙티스"
             └─→ add_documents tool
                  └─→ Q&A 쌍, 해결 사례 저장
                       └─→ Knowledge Base 확장
                            └─→ 다음 검색에 활용 🔄

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### **슬라이드 7: 명장 에이전트 - 배경 및 구축 프로세스**

#### **🎯 문제 정의**

```
자동차 제조 현장의 베테랑 마이스터(명장)들이 
은퇴를 앞두고 있으나, 수십 년간 축적된 암묵지가
체계적으로 전수되지 못하고 소실될 위기
```

#### **📚 명장의 암묵지 유형**

```
├─ 설비 이상 징후 감지 노하우
├─ 품질 불량 원인 진단 경험
├─ 생산 공정 최적화 know-how
├─ 안전사고 예방 체크리스트
└─ 문제 해결 의사결정 프로세스
```

---

#### **🔄 Phase 1: 암묵지 수집 및 구조화**

```
1️⃣ 인터뷰 및 작업 관찰
   └─→ 명장 5명 × 주 3회 × 12주
        ├─ 작업 과정 녹화 및 전사
        ├─ 의사결정 포인트 심층 인터뷰
        └─→ 약 180시간 분량의 1차 자료 수집

2️⃣ 문서화 및 정제
   └─→ 구조화된 지식 문서 작성
        ├─ 작업 표준서 보완
        ├─ 트러블슈팅 가이드
        ├─ Best Practice 사례집
        └─→ 명장 지식베이스 구축

3️⃣ 기존 자료 통합
   └─→ 과거 30년간의 현장 자료 디지털화
        ├─ 품질 불량 보고서
        ├─ 설비 이력 데이터
        ├─ 안전사고 사례
        └─→ Companion Agent에 일괄 Ingestion
```
